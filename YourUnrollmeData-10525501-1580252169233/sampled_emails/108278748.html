Hello Ambjoern,

This patch has fixes implemented for the following issues:

●

Restores from non‑root mount points with “onlyroot” option not working correctly
●

Backup retention logic had a potential to cause restore point corruption

To save current Veeam Agent settings prior to installation, save copies of the following configuration files:

●

devices.ini (/etc/veeam/devices.ini)
●

veeamdefaults.xml (/etc/veeam/veeamdefaults.xml)
●

.scp files in /etc/veeam/scripts/
●

license.lic (/etc/veeam/license/license.lic)
●

ubax.ini (/etc/veeam/ubax.ini)
●

properties.ini (/etc/veeam/properties.ini)

Prior to installing Veeam Agent v.1.0.0.41, please make sure Veeam Agent v.1.0.0.39 is uninstalled completely. After installation, replace configuration files with those you saved.

In case your configuration wasn’t saved you’d need to accept the license agreement again, install a license and configure the following if needed: backup location, backup retention, backup selection, log files location and level, and configuration file location.

In case an existing incremental backup chain consists of more than 11 points and has ever been merged due to retention policy (such chains have the .tar.gz.9 archive of zero size), a new chain should be started. Otherwise, existing backup chains can be continued.

It's recommended to re‑create recovery media after installing Veeam Agent v.1.0.0.41 so that new recovery media images include all changes.
